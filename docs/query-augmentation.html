Query Augmentation

Queries to Lettuce shouldn't return what the user asks for, but what they want. Can we augment their queries to get closer?

Currently, Lettuce queries the database first, then uses what it retrieves to inform the LLM. Perhaps it would be better to turn this on its head.

Search engines have modified user queries for a long time to improve information retrieval. Can we get an LLM to rewrite user queries so that search output is improved? Just removing irrelevant material from self-reported medications will help short-term, but this could be an invaluable approach as Lettuce handles more complex input.
